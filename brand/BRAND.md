# DeepAPI Brand System

## Risk Position

DeepAPI should not use OpenAI logos, OpenAI-style circular knot symbols,
OpenAI Sans, or copy that implies partnership, endorsement, or official
resale. The product can truthfully say it is OpenAI API compatible, but the
brand must remain visually independent.

## Identity Direction

DeepAPI uses an angular data-route mark: a forward route arrow crossing a
D-shaped gateway. This points to API routing, speed, and infrastructure rather
than a general AI lab identity.

## Assets

- `deepapi-logo.svg`: primary horizontal logo.
- `deepapi-icon.svg`: app icon or square avatar.
- `favicon.svg`: browser favicon.

## Palette

| Token | Hex | Purpose |
| --- | --- | --- |
| Night | `#08111F` | Primary background |
| Graphite | `#102033` | Panels and icon container |
| Route Green | `#24D6A5` | Primary brand signal |
| Link Blue | `#8EDBFF` | Routing accent |
| Cloud White | `#F5F8FB` | Text on dark |

## Usage

- Use the icon on dark or light backgrounds with at least 16px clear space.
- Use the full logo on landing pages, docs, invoices, and customer emails.
- Do not rotate the mark, add circular knot shapes, use OpenAI-like flower
  symbols, or pair it with OpenAI branding.
- Do not use "GPT" or "OpenAI" in the product name.

## One-API Replacement

This repository does not contain the one-api frontend source. To apply the new
identity, configure one-api's branding settings if available, or host these SVG
files publicly and set the site logo/favicon in the one-api admin UI. If the
running image does not expose branding controls, the next step is a custom
one-api frontend build or a small branded landing page in front of the gateway.
